# Project Structure Guide

This document explains the architecture of the Fraud Detection App, detailing the purpose of each directory and file, why they are necessary, and how they connect with the rest of the application.

---

## 1. `main.py`
* **What it contains:** The primary entry point for your application. It contains the code that initializes and starts your program (e.g., starting a web server or a command-line interface).
* **Why it is important:** Without this file, your app has no starting point. It acts as the "ignition switch" for the entire system.
* **Connectivity:** `main.py` imports and connects all the major components. It will pull in routes from the `api/` folder and initialize backend services from the `core/` folder.

## 2. `api/` Folder (e.g., `api/routes.py`)
* **What it contains:** Code that handles incoming requests from users or other systems. If you use a framework like FastAPI or Flask, your API endpoints (like `POST /check_fraud`) live here.
* **Why it is important:** It is the bridge between the outside world and your fraud detection logic. It receives the suspicious data (like an SMS text) and sends back the result.
* **Connectivity:** Files in this folder receive data, then pass that data to the `core/` folder for actual processing. They never do the complex math or AI tasks themselves.

## 3. `core/` Folder (e.g., `core/llm_service.py`, `core/security.py`)
* **What it contains:** The "brain" of your application. This holds your business logic, the code that communicates with AI models (like Groq or Ollama), and your custom fraud-detection rules.
* **Why it is important:** This is where the actual work happens. If you swap out a UI or an API, this core logic remains the same.
* **Connectivity:** The `core` folder receives data from the `api/` folder, processes it (perhaps by querying an LLM), and returns the results. It may also pull data schemas from the `models/` folder to ensure the data is formatted correctly.

## 4. `models/` Folder (e.g., `models/schemas.py`)
* **What it contains:** Data structures and database models. If you are using Pydantic (with FastAPI), you define what a "FraudRequest" or "User" looks like here (e.g., specifying that a phone number must be a string).
* **Why it is important:** It ensures data integrity. It prevents your app from crashing if a user sends bad data (like sending a number when text was expected).
* **Connectivity:** Both the `api/` folder and `core/` folder import these models to validate data as it moves through the system.

## 5. `data/` Folder
* **What it contains:** Local storage for things like CSV files containing past fraud records, SQLite databases, or sample text files for testing.
* **Why it is important:** Provides a dedicated, organized place for local static data so it doesn't clutter your code folders.
* **Connectivity:** Usually accessed by the `core/` folder when it needs to look up historical data or train/evaluate a local model.

## 6. `tests/` Folder
* **What it contains:** Automated test scripts (usually written with `pytest`).
* **Why it is important:** Ensures that changes to your code don't accidentally break your fraud detection logic.
* **Connectivity:** These scripts import functions from `api/`, `core/`, and `models/` and run dummy data through them to verify the outputs are correct.

## 7. `.env.example` (and `.env`)
* **What it contains:** Environment variables, such as API keys (for Groq/OpenAI), database passwords, or server ports.
* **Why it is important:** Keeps sensitive secrets out of your actual code. You should never commit a real `.env` file to GitHub, which is why we provide an `.env.example` template.
* **Connectivity:** The `main.py` or `core/` files will load these variables using a library like `python-dotenv` when the app starts.

## 8. `requirements.txt`
* **What it contains:** A list of all third-party Python libraries your project needs to run (like `fastapi`, `requests`, `python-dotenv`).
* **Why it is important:** Allows anyone else (or yourself on a new computer) to easily install all dependencies with a single command (`pip install -r requirements.txt`).
* **Connectivity:** It doesn't connect to the code directly, but dictates what libraries the code is allowed to import.

## 9. `frontend/` Folder
* **What it contains:** The HTML, CSS, and vanilla JavaScript that make up the user interface. It is designed as a Mobile App Shell.
* **Why it is important:** Provides the visual interface for users to manually scan messages, view AI agent traces, and monitor system notifications.
* **Connectivity:** Connects directly to the FastAPI backend via AJAX (`fetch()`) calls to send inputs and receive AI verdicts.

## 10. `android/` Folder
* **What it contains:** The native Android application generated by Capacitor.
* **Why it is important:** It wraps the `frontend/` into an installable `.apk` and includes the critical `NotificationListenerService` in `AndroidManifest.xml` to automatically read system-wide notifications (SMS, WhatsApp, Telegram, etc.) in the background for real-time fraud monitoring.
* **Connectivity:** Acts as a native wrapper for the web frontend, bridging web code with Android OS-level APIs.
